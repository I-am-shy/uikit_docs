## 2 示例代码运行指引

### 2.1 准备环境

请确保开发环境满足以下技术要求：

* Xcode 6.0 或以上版本
* macOS 10.11 或以上版本的 macOS 设备
* macOS 设备已经连接到 Internet

### 2.2 运行示例代码

1. 安装 Xcode: 打开 `AppStore` 搜索 `Xcode` 并下载安装。

    <img src="/Pics/iOS/ZegoExpressEngine/Common/appstore-xcode.png" width=40% height=40%>

2. 使用 Xcode 打开 `ZegoExpressExample-macOS-OC.xcodeproj`。

    打开 Xcode，点击左上角 `File` -> `Open...`

    <img src="/Pics/iOS/ZegoExpressEngine/Common/xcode-open-file.png" width=70% height=70%>

    找到第一步下载解压得到的示例代码文件夹中的 `ZegoExpressExample-macOS-OC.xcodeproj`，并点击 `Open`。

3. 登录 Apple ID 账号。

    打开 Xcode, 点击左上角 `Xcode` -> `Preference`，选择 `Account` 选项卡，点击左下角的 `+` 号，选择添加 Apple ID。

    <img src="/Pics/iOS/ZegoExpressEngine/Common/xcode-account.png" width=90% height=90%>

    输入 Apple ID 和密码以登录。

    <img src="/Pics/iOS/ZegoExpressEngine/Common/xcode-login-apple-id.png" width=70% height=70%>

4. 修改开发者证书和 Bundle Identifier。

    打开 Xcode，点击左侧的项目 `ZegoExpressExample-macOS-OC`

    <img src="/Pics/iOS/ZegoExpressEngine/Common/xcode-select-project.png" width=50% height=50%>

    在 `General` 选项卡中，更改 `Bundle Identifier`（可修改为 `com.your-name.ZegoExpressExample-macOS-OC`）

    <img src="/Pics/iOS/ZegoExpressEngine/Common/bundle-identifier.png" width=90% height=90%>

    点击 `Signing & Capabilities` 选项卡，在 `Team` 中选择自己的开发者证书

    <img src="/Pics/iOS/ZegoExpressEngine/Common/team-signing.png" width=90% height=90%>

5. 下载的示例代码中缺少 SDK 初始化必须的 AppID 和 AppSign，请参考 [获取 AppID 和 AppSign 指引 \|_blank](https://doc.zego.im/API/HideDoc/GetExpressAppIDGuide/GetAppIDGuideline.html) 获取 AppID 和 AppSign。如果没有填写正确的 AppID 和 AppSign，源码无法正常跑起来，所以需要修改 `ZegoExpressExample-macOS-OC/Helper` 目录下的 `ZGKeyCenter.m`，填写正确的 AppID 和 AppSign。

    <img src="/Pics/iOS/ZegoExpressEngine/Common/appid-appsign.png" width=80% height=80%>

6. 点击 Xcode 左上角的 Build 按钮进行编译和运行。

    <img src="/Pics/iOS/ZegoExpressEngine/Common/build-and-run.png" width=50% height=50%>

## 3 常见问题

1. `The app ID "im.zego.ZegoExpressExample-macOS-OC" cannot be registered to your development team. Change your bundle identifier to a unique string to try again.`

    参考上面的 **步骤2.2-4 修改开发者证书和 Bundle Identifier**，在 `Targets` -> `Signing & Capabilities` 中切换为自己的开发证书并修改 `Bundle Identifier` 后再运行。
