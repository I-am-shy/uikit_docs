# 媒体播放器

- - -

## 1 功能简介

媒体播放器组件提供播放音视频媒体文件的能力，并且支持将播放的媒体文件的音画数据推流出去。

### 1.1 应用场景

- 播放测试音频：可以使用媒体播放器播放测试音频，验证音频播放设备是否工作正常。
- 播放背景音乐：使用媒体播放器播放音乐并混在流中推送出去，让远端听到背景音乐。
- 播放视频文件：结合自定义视频采集将媒体资源的视频数据推送出去，远端可拉流观看。

### 1.2 支持格式

媒体播放器支持 MP3、MP4、FLV、WAV、AAC、M3U8 等格式文件。

<div class = 'mk-hint'>

媒体播放器默认支持格式为：MP3、MP4、FLV、WAV、AAC、M3U8。如需支持其它格式，请联系 ZEGO 技术支持。
</div>

### 1.3 支持协议

支持 HTTP、HTTPS 和 RTMP 协议。

## 4 使用步骤

### 4.1 创建媒体播放器

调用 [IZegoExpressEngine\|_blank](@-IZegoExpressEngine) 的 [createMediaPlayer \|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-express-engine#create-media-player) 接口以创建媒体播放器实例。一个媒体播放器实例只能播放一个音视频，引擎同一时间最多支持创建 4 个播放器，以达到同时播放多个文件的效果。超出 4 个则创建失败，接口将返回 `nullptr`。

```C++
IZegoMediaPlayer* mediaPlayer = engine->createMediaPlayer();
if (mediaPlayer == nullptr) {
    printf("创建播放器失败");
}
```


### 4.2 （可选）为播放器设置事件回调

<details class="zg-primary">
    <summary>播放器事件回调设置</summary>

调用媒体播放器的 [setEventHandler \|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#set-event-handler) 接口为播放器设置事件回调 [IZegoMediaPlayerEventHandler\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player-event-handler)，以接收“播放器状态改变”、“播放进度改变”、“播放器网络状态更新”等通知。

```C++
class MyMediaPlayerEventHandler: public IZegoMediaPlayerEventHandler{
public:
    // 播放器播放状态变更及事件回调
    void onMediaPlayerStateUpdate(IZegoMediaPlayer* mediaPlayer, ZegoMediaPlayerState state, int errorCode) override{
        printf("currentState: %d", state);
    }

    // 播放器网络状态事件回调
    void onMediaPlayerNetworkEvent(IZegoMediaPlayer* mediaPlayer, ZegoMediaPlayerNetworkEvent networkEvent) override {
        printf("networkEvent: %d", networkEvent);
    }

    // 播放器播放进度回调
    void onMediaPlayerPlayingProgress(IZegoMediaPlayer* mediaPlayer, unsigned long long millisecond) override {
        printf("currentProgress:%lld", millisecond);
    }
}

auto eventHandler = std::make_shared<MyMediaPlayerEventHandler>();
mediaPlayer->setEventHandler(eventHandler);
```

</details>



### 4.3 加载媒体资源

调用媒体播放器的 [loadResource\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#load-resource) 来指定要播放的媒体资源。可以是本地资源的绝对路径，如 “D:/Zego/your-movie.mp4”；也可以是网络资源的 URL，如 “http://your.domain.com/your-movie.mp4”。用户可通过传入回调参数的方式获取加载文件的结果。

<div class = 'mk-warning'>

如果正在播放之前加载的媒体文件，请先调用 [stop\|_blank](@stop-IZegoMediaPlayer) 接口停止播放，然后再调用 [loadResource\|_blank](@loadResource-IZegoMediaPlayer) 接口加载媒体资源，否则无法加载成功。
</div>

```C++
mediaPlayer->loadResource("sourcePath", [=](int errorCode){
    if(errorCode == 0){
        printf("the total duration of the file: %lld", mediaPlayer->getTotalDuration());
    } else {
        print("load resource failed");
    }
});
```

### 4.4 播放控制

#### 4.4.1 播放状态控制

<img src="/Pics/Common/ZegoExpressEngine/mediaplayer_state.png">

调用 [start\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#start)、[pause\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#pause)、[resume\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#resume)、[stop\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#stop) 来启停播放。一旦播放器的内部状态改变，[onMediaPlayerStateUpdate\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player-event-handler#on-media-player-state-update) 回调将会被触发。

用户也可通过调用 [getCurrentState\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#get-current-state) 随时获取播放器的当前状态。

如果 [enableRepeat\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#enable-repeat) 设置为 “true”，则播放器会在播放完文件后自动重播。

```C++
mediaPlayer->enableRepeat(true);
mediaPlayer->start();
mediaPlayer->pause();
mediaPlayer->resume();
mediaPlayer->stop();
```

#### 4.4.2 播放进度控制

播放文件的进度会通过 [onMediaPlayerPlayingProgress\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player-event-handler#on-media-player-playing-progress) 方法回调，默认触发回调的间隔是 1000 ms，可通过 [setProgressInterval\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#set-progress-interval) 来更改此间隔。

用户也可通过 [getCurrentProgress\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#get-current-progress) 来获取当前播放进度。

通过 [seekTo\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#seek-to) 接口来调整进度。

```C++
mediaPlayer->setProgressInterval(2000);
auto progress = mediaPlayer->getCurrentProgress();
mediaPlayer->seekTo(mediaPlayer->getTotalDuration() / 2, nullptr);
```

#### 4.4.3 播放器音频控制

通过 [getPlayVolume \|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#get-play-volume) 和 [setPlayVolume \|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#set-play-volume) 获取和控制播放音量。

通过 [getPublishVolume \|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#get-publish-volume) 和 [setPublishVolume \|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#set-publish-volume) 获取和控制推流音量。

调用 [enableAux \|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#enable-aux) 可以将文件的声音混入正在推的流中。

调用 [muteLocal \|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#mute-local) 可以控制本地静音播放。

```C++
auto playVolume = mediaPlayer->getPlayVolume();
mediaPlayer->setPlayVolume(playVolume / 2);
mediaPlayer->muteLocal(true);
mediaPlayer->enableAux(true);
```

如果想获取文件的音频数据，可通过 [setAudioHandler\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player#set-audio-handler) 接口来为播放器指定音频帧回调 [IZegoMediaPlayerAudioHandler\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-media-player-audio-handler)。

```C++
/// 播放器抛音频数据的回调
class MyAudioHandler: public IZegoMediaPlayerAudioHandler {
public:
    virtual void onAudioFrame(IZegoMediaPlayer* /*mediaPlayer*/, const unsigned char* /*data*/, unsigned int /*dataLength*/, ZegoAudioFrameParam /*param*/) {
    }
};
auto audioHandler = std::make_shared<MyAudioHandler>();
mediaPlayer->setAudioHandler(audioHandler);
```

#### 4.4.4 播放器视频控制

当播放视频资源时，用 [setPlayerCanvas\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Video_SDK_API~CPP~class~zego-express-i-zego-media-player#set-player-canvas) 来设置视频的显示视图。

```C++
HWND playWND = ...; // 填写windows窗口句柄
ZegoCanvas canvas(playWND);
mediaPlayer->setPlayerCanvas(&canvas);
```

如果想获取文件的视频数据，可通过 [setVideoHandler\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Video_SDK_API~CPP~class~zego-express-i-zego-media-player#set-video-handler) 接口来为播放器指定视频帧回调 [IZegoMediaPlayerVideoHandler\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Video_SDK_API~CPP~class~zego-express-i-zego-media-player-video-handler)。

```C++
/// 播放器抛视频数据的回调
class MyVideoHandler: public IZegoMediaPlayerVideoHandler {
public:
    void onVideoFrame(IZegoMediaPlayer* /*mediaPlayer*/, const unsigned char** /*data*/, unsigned int* /*dataLength*/, ZegoVideoFrameParam /*param*/) {
    }
};
auto videoHandler = std::make_shared<MyVideoHandler>();
mediaPlayer->setVideoHandler(videoHandler);
```

#### 3.4.5 将播放器播放的视频推流出去

1. 将播放器的视频推流出去前，需要先通过 [setVideoHandler\|_blank](/zh/api?doc=Express_Video_SDK_API~CPP~class~zego-express-i-zego-media-player#set-video-handler) 设置视频帧回调监听，用于获取 [onVideoFrame\|_blank](/zh/api?doc=Express_Video_SDK_API~CPP~class~zego-express-i-zego-media-player-video-handler#on-video-frame) 抛出的视频帧数据。

2. 使用自定义方式采集视频，并将获取到的视频数据混入推流数据中，详细操作请参考 [视频进阶 - 自定义视频采集 \|_blank](#3678)。

<div class= 'mk-hint'>

自定义采集数据时，建议开发者自行定义一个 “flag” 标记位：

- 当触发 [onStart\|_blank](/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-custom-video-capture-handler#on-start) 回调时将 “flag” 标记设置为 “True”，表示可以开始将自定义采集的视频数据发送给 SDK。
- 当触发 [onStop\|_blank](/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-custom-video-capture-handler#on-stop) 回调时将 “flag” 标记设置为 “False”，表示需要停止发送采集的视频数据给 SDK。
</div>

3. 开发者需要在 [onVideoFrame\| _blank](/zh/api?doc=Express_Video_SDK_API~CPP~class~zego-express-i-zego-media-player-video-handler#on-video-frame) 中添加对 “flag” 的判断逻辑，当 “flag” 设置为 “True” 时（即触发了 [onStart\|_blank](/zh/api?doc=Express_Video_SDK_API~CPP~class~zego-express-i-zego-custom-video-capture-handler#on-start) 回调），调用 [sendCustomVideoCaptureRawData\|_blank](/zh/api?doc=Express_Video_SDK_API~CPP~class~zego-express-i-zego-express-engine#send-custom-video-capture-raw-data) 方法向 SDK 发送已获取的视频数据。

4. 调用 [startPublishingStream\| _blank](/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-express-engine#start-publishing-stream) 开始推流，请参考 [快速开始 - 实现流程\|_blank](7633#3_3) 的 “3.3 推流”。



#### 4.4.6 变声

处理类似于 KTV 中对伴奏升降调等场景时，可以调用媒体播放器的 [setVoiceChangerParam \|_blank](/zh/api?doc=Express_Audio_SDK_API~CPP_windows~class~zego-express-i-zego-express-engine#set-voice-changer-param) 接口来实现变声功能。开发者可通过 [ZegoVoiceChangerParam \|_blank](/zh/api?doc=Express_Audio_SDK_API~CPP~struct~zego-express-zego-voice-changer-param) 对象中的音高参数 “pitch” 设置变声效果，该参数取值范围为 [-8.0, 8.0]，值越大声音越尖锐，默认值为 “0.0”（即关闭变声器）。

```c++
ZegoVoiceChangerParam param = ZegoVoiceChangerParam();
param.pitch = 8.0f;     /** 男声变童声 */
param.pitch = 4.0f;     /** 男声变女声 */
param.pitch = 6.0f;     /** 女声变童声 */
param.pitch = -3.0f;    /** 女声变男声 */
mediaPlayer->setVoiceChangerParam(ZEGO_MEDIA_PLAYER_AUDIO_CHANNEL_ALL, param);
```

### 4.5 销毁媒体播放器

使用完播放器之后，需要及时的调用 [destroyMediaPlayer\|_blank](https://doc-zh.zego.im/zh/api?doc=Express_Audio_SDK_API~CPP~class~zego-express-i-zego-express-engine#destroy-media-player) 接口以释放该播放器占用的资源。

```C++
engine->destroyMediaPlayer(mediaPlayer);
```
