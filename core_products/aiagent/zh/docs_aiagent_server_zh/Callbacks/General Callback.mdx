
# 通用回调

- - -

<Warning title="注意">回调服务不能保证完全可靠，请慎重考虑使用回调方案构建核心业务流程的风险。</Warning>

## 描述

凭借本回调，您可以监听用户与智能体对话过程中所发生的事件，包含 ASR 结果、LLM 结果、异常事件。

<Note title="说明">

如需获取 ASR 结果和 LLM 结果，您需要在 [创建智能体实例](./../Agent%20Instance%20Management/CreateAgentInstance.mdx) 时，将 `ChatConfig.EnableASRServerMessage` 和 `ChatConfig.EnableLLMServerMessage` 设置为 `1`。
</Note>
## 回调说明

- 请求方法：POST。
    <Note title="说明">回调数据格式为 JSON。您需要对其进行 UrlDecode 解码.</Note>
- 请求地址：请联系 ZEGO 技术支持配置。
- 传输协议：HTTPS/HTTP，建议使用 HTTPS。

## 回调参数

<Note title="说明">回调的相关参数，ZEGO 会在之后的迭代计划中，持续优化更新（例如：新增字段、或新增某些字段的参数取值）。开发者在接入时，请避免将代码写死，造成后期更新后，无法兼容新版本。</Note>

| 参数 | 类型 | 描述 |
|------|------|------|
| AppId | Number | ZEGO 给开发者 APP 的唯一标识。 |
| EventType | Number | 事件通知类型。<ul><li>203：回调 ASR 结果。</li><li>204：回调 LLM 结果。</li><li>301：异常事件。</li></ul> |
| Nonce | String | 随机数，用于检验串计算。 |
| Timestamp | Number | 回调发送时的 Unix 时间戳（秒），用于检验串计算。 |
| Signature | String | 检验串，验证回调发送方身份。 |
| AgentId          | String | 智能体的唯一标识。   |
| AgentInstanceId  | String | 智能体实例的唯一标识。 |
| Data | Object | 事件详细信息。本参数结构，请参考 [Data](#data)。 |

### Data

**根据 `EventType` 取值不同，Data 包含的参数不同。**

<Tabs>
<Tab title="203">
| 参数 | 类型 | 描述 |
|------|------|------|
| Text | String | ASR 结果文本。 |
</Tab>
<Tab title="204">
| 参数 | 类型 | 描述 |
|------|------|------|
| Round | Number | LLM 结果所属的对话轮次，每次用户主动说话，轮次都会增加（有序，非递增）。 |
| Text | String | LLM 输出文本。 |
</Tab>
<Tab title="301">
| 参数 | 类型 | 描述 |
|------|------|------|
| Code | Number | 事件码：<ul><li>1001：ZEGO 实时互动 AI Agent 通用错误，详情请参考 `Message` 字段。</li><li>1004：服务端 VAD 检测到人声说话但是 ASR 没有识别有效文本。</li><li>1101：IM 通用错误，请向 ZEGO 技术支持提供 `Message` 字段。</li><li>1201：RTC 通用错误，请向 ZEGO 技术支持提供 `Message` 字段。</li><li>2101：ASR 通用错误，请向 ZEGO 技术支持提供 `Message` 字段。</li><li>2201：LLM 通用错误，详情请参考 `Message` 字段。</li><li>2202：LLM 参数错误，请检查请求参数是否合法。</li><li>2203：LLM 请求鉴权失败，请检查传入的 `LLM.ApiKey` 是否正确。</li><li>2204：LLM 请求并发达到上限，请确认是否增购并发。</li><li>2205：LLM 账号欠费，请您自行前往 LLM 服务提供商官网续费。</li><li>2206：LLM 内容审核失败，请检查用户是否输入非法内容。</li><li>2301：TTS 通用错误，详情请参考 `Message` 字段。</li><li>2302：TTS 参数错误，请检查请求参数是否合法。</li><li>2303：TTS 请求鉴权失败，请检查传入的 `TTS.Params.app` 是否正确。</li><li>2304：TTS 请求并发达到上限，请确认是否增购并发。</li><li>2305：TTS 账号欠费，请您自行前往 TTS 服务提供商官网续费。</li></ul> |
| Message | String | 异常信息描述。 |
</Tab>
</Tabs>

## 返回响应

当您收到回调后，请返回 HTTP status code 为 2XX （例如 200），表示接收成功。返回其他，都表示接收失败。

## 回调重试策略

如果 ZEGO 服务器没有收到响应，或收到的 HTTP status code 不为 2XX（例如 200），都会尝试重试，**最多进行 5 次重试**。每次重试请求与**上一次请求**的间隔时间分别为 2s、4s、8s、16s、32s。若第 5 次重试后仍然失败，将不再重试，该回调丢失。
