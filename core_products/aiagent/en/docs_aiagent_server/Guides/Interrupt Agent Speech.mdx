# Interrupt Agent

Different scenarios require different ways to interrupt the agent's speech. Currently, two interruption methods are supported and can be used in combination.

| Interruption Method | Description |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Voice Interruption | When voice interruption is enabled, the agent continuously monitors the user's speech status and recognizes speech content while speaking. If the user starts speaking, it will interrupt and stop the agent's speech (stopping the current LLM request and TTS request) and start the next round of response. If voice interruption is disabled, the next round will only begin after the agent finishes speaking: monitoring user's speech status and content, then starting to respond. |
| Manual Interruption | Through API, you can directly interrupt the agent's current content |

## Common Scenarios

| Voice Call Mode | Interruption Combination | Common Scenarios |
| ---------------------------- | --------------------------------- | -------------------------------------------------------------------- |
| Natural Voice Chat | ✅Enable Voice Interruption<br/>❌No Manual Interruption | AI Virtual Companion<br/>AI Voice Assistant<br/>AI Customer Service |
| Push-to-talk Mode | ❌Disable Voice Interruption<br/>✅Use Manual Interruption | Noisy Exhibition Environment<br/>Brief Speeches in Seminars<br/>Time-limited Speaking in Games like Werewolf |

## Implementation Steps

### Enable or Disable Voice Interruption

When [creating an agent instance](./../API%20reference/Agent%20Instance%20Management/Create%20Agent%20instance.mdx), you can control voice interruption by setting the `AdvancedConfig.InterruptMode` parameter.

| Parameter | Type | Required | Description |
|----------------|------|------|----------------------------------------------------------------------|
| InterruptMode | Number | No | Mode for voice interruption during agent's response: <ul><li>0: Voice interruption enabled</li><li>1: Voice interruption disabled (ASR starts after AI output (TTS playback) completes)</li></ul>Default is 0 if not set. |

### Manual Interruption

Call the interrupt agent instance status interface ([InterruptAgentInstance](./../API%20reference/Agent%20Instance%20Control/Interrupt%20Agent%20Instance.mdx)) with the `AgentInstanceId` returned from the [Create Agent Instance](./../API%20reference/Agent%20Instance%20Management/Create%20Agent%20instance.mdx) interface to immediately interrupt the agent's speech.

## Receive Agent Interruption Event Notifications
<Steps>
<Step title="Configure Callback Address">
Contact ZEGOCLOUD technical support to configure the address for receiving AI Agent backend callbacks.
</Step>
<Step title="Enable Callback Reception">
When [creating an agent instance](./../API%20reference/Agent%20Instance%20Management/Create%20Agent%20instance.mdx), set the request parameter `CallbackConfig.Interrupted` to 1.
</Step>
<Step title="Receive Callbacks">
When the agent is interrupted, the AI Agent backend will send an interruption event notification (`Event` is `Interrupted`) to the configured address. Here's an example:
```json {7}
{
    "AppId": 123456789,
    "Nonce": "abcdd22113",
    "Timestamp": 1747033950524，
    "Sequence": 1921825797275873300,
    "Signature": "XXXXXXX",
    "Event": "Interrupted",
    "RoomId": "90000001237",
    "AgentInstanceId": "1921825671047294976",
    "AgentUserId": "apitest689_agent",
    "Data": {
        "Round": 1481651956,
        "Reason": 1
    }
}
```
The `Reason` parameter is explained below:
| Parameter | Type | Description |
|--------|--------|--------------|
| Reason | Number | Interruption reason: <ul><li>1: User is speaking.</li><li>2: You actively called [LLM](./../API%20reference/Agent%20Instance%20Control/Send%20Agent%20instance%20LLM.mdx) on the server.</li><li>3: You actively called [TTS](./../API%20reference/Agent%20Instance%20Control/Send%20Agent%20instance%20TTS.mdx) on the server.</li><li>4: You actively [interrupted the agent instance](./../API%20reference/Agent%20Instance%20Control/Interrupt%20Agent%20Instance.mdx) on the server.</li></ul> |
</Step>
</Steps>
