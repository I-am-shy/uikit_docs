# API overview


ZEGOCLOUD's In-app Chat provides you with the following Server APIs, please refer to [Accessing Server APIs\|_blank](!zim-Making_API_Requests) to call the following interface.

## User related

| Method | Description | Default Frequency Limit |
|-|-|-|
|[Query user's online status\|_blank](!query_User_Online_State)| Query the online status of a single user or multiple users.| 20 times/second |
|[Batch register users\|_blank](!zim-server_user_Registration)| Register ZEGOCLOUD Instant Messaging (ZIM) accounts for users.| 20 times/second |
|[Modify user information\|_blank](!zim-ModifyUserInfo)| Modify user information, including the nickname and avatar.| 20 times/second |
|[Batch add friends\|_blank](!zim-AddFriends)| Add up to 20 friends for a user without their consent.| 20 times/second |
|[Batch send friend requests\|_blank](!zim-SendFriendApplication)| Send friend requests to up to 20 users for a user.| 20 times/second |
|[Batch delete friends\|_blank](!zim-DeleteFriends)| Delete up to 20 friends for a user in a one-way or two-way manner.| 20 times/second |
|[Delete all friends\|_blank](!zim-DeleteAllFriends)| Delete all friends for a user in a one-way or-way manner.| 20 times/second |
|[Query the friend list\|_blank](!zim-QueryFriendList)| Query the list of friends by page based on a user ID, including information such as the friend alias and friend relation creation time.| 20 times/second |
|[Check friendship\|_blank](!zim-CheckFriendsRelation)| Check the friend relation between a user and up to 20 users.| 20 times/second |
|[Change the alias of a friend\|_blank](!zim-UpdateFriendsAlias)| Change the aliases of up to 20 friends for a user.| 20 times/second |
|[Modify the attributes of a friend\|_blank](!zim-UpdateFriendAttributes)| Modify the attributes of a friend for a user.| 20 times/second |
|[Batch block users\|_blank](!zim-AddUsersToBlacklist)|Block up to 20 users for a user.| 20 times/second |
|[Batch unblock users\|_blank](!zim-RemoveUsersFromBlacklist)| Unblock up to 20 users for a user.| 20 times/second |
|[Query the blocklist\|_blank](!zim-QueryBlacklist)| Get the list of all blocked friends based on a user ID by page.| 20 times/second |
|[Check blockships\|_blank](!zim-CheckUsersIsInBlacklist)|  Check whether up to 20 users are on the target user's blacklist.| 20 times/second |

| Method | Description | Default Frequency Limit|
|-|-|-|

<tbody><tr>
<th>Callback</th>
<th>Description</th>
</tr>
<tr>
<td>[Login and logout callback|_blank](!zim-Login_Logout_Callback)</td>
<td>Listen for user login and logout and implement the business logic, for example, counting online users.</td>
</tr>
<tr>
<td>[Callback on call invitation sent|_blank](!zim-third_party_call_create)</td>
<td>After a call is created on the client, the business backend receives the call creation callback from the ZIM server to check whether the call is created successfully.</td>
</tr>
<tr>
<td>[Callback on call invitation canceled|_blank](!zim-Third_party_call_cancel)</td>
<td>When the following situations occur, the business backend can receive a callback from the ZIM server to cancel a call, which helps determine the call status:<ul><li>The initiating user successfully cancels the call.</li><li>The initiating user proactively logs out or enters a logged-out state due to a timeout after successfully creating the call.</li></ul></td>
</tr>
<tr>
<td>[Callback on call invitation accepted|_blank](!zim-call_accept)</td>
<td>After the callee accepts the call invitation, the business server can receive the callback on the call invitation accepted by the In-app Chat Server to determine the success of the call invitation acceptance request.</td>
</tr>
<tr>
<td>[Callback on call invitation rejected|_blank](!zim-call_reject)</td>
<td>After the callee rejects the call invitation, the business server can receive the callback on the call invitation rejected by the In-app Chat Server to determine the success of the call invitation request.</td>
</tr>
<tr>
<td>[Callback on call invitation timed out|_blank](!zim-call_timeout)</td>
<td>After the call invitation is sent successfully, when the following situations occur, the business server can receive the call timeout callback from the In-app Chat Server to know the target user:<ul><li>The callee has not answered until the timeout period configured in the business server is met.</li><li>The callee does not answer until the timeout period set when the call was created is met.</li></ul></td>
</tr>
<tr>
<td>[Callback on message not sent yet|_blank](!zim-callback_before_message_sent)</td>
<td>After setting this callback, when a user sends a message to a one-on-one chat, group chat or room chat, In-app Chat will initiate a request to your business server, and you can perform real-time operations on the message through the response.</td>
</tr>
<tr>
<td>[Callback on message sent|_blank](!zim-zim_send_msg)</td>
<td>After the user sends a message to one-on-one chat, group chat, or room successfully or fails, the business server can receive the message sending callback from the In-app Chat server, and synchronize the message sent by the user to the business server in real time, and store it in the business server.</td>
</tr>
</tbody></table>
