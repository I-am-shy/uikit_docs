# 自定义点击跳转


## 描述

通常，用户点击离线推送通知时，会跳转到应用首页。基于本功能，开发者可以自定义离线推送通知的点击跳转逻辑，比如用户在点击推送消息时，能够跳转到指定页面，从而提升用户体验和交互效率。

## 前提条件

在实现自定义点击跳转之前，请确保：
- 已集成 2.8.0 或以上版本的 ZPNs SDK，详情请参考 [实现离线推送](./Implement%20offline%20push%20notification.mdx)。
- 已在控制台配置离线推送证书，详情请参考控制台文档 [服务配置 - 即时通讯 - 离线推送配置](https://doc-zh.zego.im/article/16233)。

## 实现流程

<Steps>
<Step title="点击推送通知，进入自定义的应用内页">

参考 https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/push-send-alert#section8794131614597 在工程内配置 uris 参数用于点击消息进入应用内页。


在控制台配置click_type 为 1（打开应用自定义页面），url 为工程内配置的 url 的 resourceID.

发送消息时，在 ZIMPushConfig 中携带该 resourceID 即可。
