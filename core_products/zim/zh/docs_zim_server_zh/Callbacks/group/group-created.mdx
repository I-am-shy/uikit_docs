# 创建群组

---

## 描述

凭借本回调，您可以实时监听用户创建群聊的行为，查看用户创建的群组的信息

## 回调说明

- 请求方法：POST。
  <Note title="说明">
  回调数据格式为 JSON，您需要对其进行 UrlDecode 解码。
  </Note>
- 请求地址：请在 [ZEGO 控制台](https://console.zego.im/) 上配置回调地址，配置流程请参考控制台文档 [ZIM 相关回调配置](https://doc-zh.zego.im/article/17223)。
- 传输协议：HTTPS/HTTP，建议使用 HTTPS。

## 回调参数

| 公共参数 | 类型 | 说明 |
|------|------|------|
| appid | String | App 的唯一标识。 |
| event | String | 回调事件，此回调返回值为 `create_group`。 |
| timestamp | Int | 创建群的时间，以服务器当前时间记录，Unix 时间戳，单位为秒（s）。 |

| 业务参数 | 类型 | 说明 |
|------|------|------|
| group_id | String | 群组 ID。 |
| operator_account | String | 群组创建操作者的用户 ID。 |
| owner_account | String | 群主 ID。 |
| group_name | String | 群名称。 |
| attributes | Arrays | 群自定义的属性。 |
| user_ids | Arrays | 创建群组时，所有群成员的用户 ID。 |

## 数据示例

- POST/JSON

```json
{
    "appid" : "1",
    "event" : "create_group",
    "group_id" : "1",
    "operator_account" : "userA",
    "owner_account" : "operator_account",
    "group_name" : "group_name",
    "attributes" : [
        {
            "Key" : "UserSet1",
            "Value" : "UserGet1",
            "Seq" : "1"
        }
        {
            "Key" : "UserSet2",
            "Value" : "UserGet2",
            "Seq" : "2"
        }
    ],
    "timestamp" : 1499676978,
    "user_ids" : ["Tony","Jenny"]
}
```

## 返回响应

返回 HTTP status code 为 2XX （例如 200）表示成功，其他响应都表示失败。

## 回调重试策略

如果 ZEGO 服务器没有收到响应，或收到的 HTTP status code 不为 2XX（例如 200），都会尝试重试，**最多进行 5 次重试**。每次重试请求与**上一次请求**的间隔时间分别为 2s、4s、8s、16s、32s。若第 5 次重试后仍然失败，将不再重试，该回调丢失。
