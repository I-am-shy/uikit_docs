

# 查询数字人补帧包

---

## 描述

当您需要将数字人视频 A 与视频 B 拼接为一段视频时，可能会因为视频 A 结尾的动作与视频 B 开头的动作不同，导致拼接后的视频动作不连贯。您可以调用本接口查询获取数字人补帧包，补帧包内含该数字人形象的若干段动作视频。

您需要根据 [查询视频合成任务\|_blank](#18548) 获取的视频 A 和 B 的 `StartIndex`，如 775 和 1550，在补帧包中查看文件名为 775-1550 的视频，以便用于视频 A 与视频 B 之间的动作衔接。

<Note title="Note">


如需调用此接口，请先发送邮件至 [aigc@zego.im](mailto:aigc@zego.im) 进行配置。
</Note>


## 接口原型

- 请求方法：POST
- 中国大陆正式环境请求地址：https://aigc-api.zegotech.cn/?Action=DescribeMetaHuman
- 传输协议：HTTPS
- 调用频率限制：10 次/秒


## 请求参数

以下请求参数列表仅列出了接口请求参数，公共参数列表请参考 [调用方式 - 公共请求参数\|_blank](#18534#4_1)。

| 参数        | 类型   | 是否必选 | 描述            |
| :---------- | :----- | :------- | :-------------- |
| MetaHumanId | String | 是       | 数字人形象 ID，通过 [查询数字人模特列表\|_blank](!DescribeMetaHumanModel) 获得。 |

## 请求示例

- 请求 URL  

    ```
    https://aigc-api.zegotech.cn/?Action=DescribeMetaHuman
    &AppId=1234567890
    &SignatureNonce=15215528852396
    &Timestamp=1234567890
    &Signature=7a2c0f11145fb760d607a07b54825013
    &SignatureVersion=2.0
    ```

- 请求消息体

    ```json
    {
        "MetaHumanId": "your_metahuman_id"
    }
    ```
