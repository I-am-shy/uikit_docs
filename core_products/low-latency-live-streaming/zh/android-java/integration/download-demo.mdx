## 示例源码运行指引

@@@SupTestEnvNewDocWarn@@@

本文介绍如何快速跑通示例源码，体验基础的音视频通话服务。

### 1 准备环境

在开始集成 ZEGO Express SDK 前，请确保开发环境满足以下要求（以下说明皆以 macOS 开发电脑为例)：

@@@Android_Studio_release_notes@@@

* Android SDK 30、Android SDK Platform-Tools 30.x.x 或以上版本。
* Android 4.4 或以上版本，且支持音视频的 Android 设备或模拟器（推荐使用真机），如果是真机，请开启“允许调试”选项。
* Android 设备、macOS 开发电脑已经连接到 Internet。

<div class = 'mk-hint'>

- 首次编译示例源码时，如果没有对应的依赖库或者构建工具，Android Studio 会自动下载示例源码，可能需要较长时间，请耐心等待。
- 建议下载特定的 Gradle 及所需的依赖库，请参考 [Gradle Services\|_blank](https://services.gradle.org) 下载。示例源码中使用的 Gradle 相关版本如下：
  - Gradle：6.7.1（在 “project/gradle/wrapper/gradle-wrapper.properties” 文件中查看）。
  - Android Gradle 插件：4.2.0（在 “project/build.gradle” 文件中查看）。

> 关于 Android Gradle 插件、Gradle、SDK Tool 之间的版本依赖关系，请查看：[Android Gradle 插件版本说明\|_blank](https://developer.android.com/studio/releases/gradle-plugin)

</div>

### 3 运行示例源码

1. 在开发电脑中找到 Android Studio 软件图标。
   <img src="/Pics/Android/ExpressSDK/SampleRunningGuide/open_as.png">

  <details class="zg-primary">   
    <summary>在开发电脑上安装 Android Studio 的方式</summary> 

    - 方式 1：登录 [Android Studio 官网\|_blank](https://developer.android.com/studio) 下载并安装 Android Studio 及其相关的 SDK Tools。由于国内运营商网络限制等原因，开发者需要确保本机能连接该网站，若无法访问建议选择方式 2。
    - 方式 2：使用搜索引擎搜索 “Android Studio 下载”、“Android Studio 安装” 和 “Android Studio SDK Tools 安装”，查找可下载的源网站，并安装相关软件和工具。
  </details>


2. 打开 Android Studio，单击 “Open an Existing Project”。
   <img src="/Pics/Android/ExpressSDK/SampleRunningGuide/as_pre_ui.png">

3. 在解压后的示例源码文件夹中，选择 ZegoExpressExample 文件夹，并单击 “Open” 打开。
![/Pics/Android/ExpressSDK/SampleRunningGuide/open_express_android_demo_new.png](http://doc.oa.zego.im/Pics/Android/ExpressSDK/SampleRunningGuide/open_express_android_demo_new.png)

4. 示例源码中缺少 SDK 初始化所需的 “appID”、“userID” 和 “token”，需要修改 “./KeyCenter/src/main/java/im/zego/keycenter” 文件，请使用本文 [前提条件](!DownloadDemo/DownloadDemo#1_2) 已获取的 AppID 和 Token 正确填写（**注意：需要在 AppID 后加 L**），否则示例源码无法正常运行。


```java
package im.zego.keycenter;

public final class KeyCenter {

    // Developers can get appID from admin console.
    // https://console.zego.im/dashboard
    // for example:
    //     private long _appID = 123456789L;
    private long _appID = ;

    // Developers should customize a user ID.
    // for example:
    //     private String _userID = "zego_benjamin";
    private String _userID = "";

    // Developers can get token from admin console.
    // https://console.zego.im/dashboard
    // Note: The user ID used to generate the token needs to be the same as the userID filled in above!
    // for example:
    //     private String _token = "04AAAAAxxxxxxxxxxxxxx";
    private String _token = "";

    private static KeyCenter instance = new KeyCenter();
    private KeyCenter() {}

    public static KeyCenter getInstance() {
        return instance;
    }

    public long getAppID() {
        return _appID;
    }

    public void setAppID(long appID) {
        _appID = appID;
    }

    public String getUserID() {
        return _userID;
    }

    public void setUserID(String userID) {
        _userID = userID;
    }

    public String getToken() {
        return _token;
    }

    public void setToken(String token) {
        _token = token;
    }

}
```

5. 将 Android 设备链接到开发电脑，当该设备成功开启“开发者模式”和“USB 调试”功能后，可以看到 Android Studio 上方的 “Running Devices” 选项框由下图：
    ![/Pics/Android/ExpressSDK/SampleRunningGuide/as_no_device_new.png](http://doc.oa.zego.im/Pics/Android/ExpressSDK/SampleRunningGuide/as_no_device_new.png)
   变为下图：
    ![/Pics/Android/ExpressSDK/SampleRunningGuide/as_connected_device_new.png](http://doc.oa.zego.im/Pics/Android/ExpressSDK/SampleRunningGuide/as_connected_device_new.png)    
   即 Android Studio 已成功连接到 Android 设备，可以将示例源码运行到该设备上了。
    
    <div class = 'mk-hint'>

    - 当开发电脑连接 Android 设备后，大部分设备会弹出提示，不同 Android 厂商的提示信息有所不同，可根据情况选择 USB 模式。
    - 当 Android 设备连接开发电脑后，需要根据情况在该设备上开启开发者模式和 USB 调试功能（不同厂商的 Android 设备开启方式不同）。具体开启方式开发者可以需要自行搜索引擎搜索，例如：目前存在一台未开启过开发者模式与 USB 调试功能的某设备，可以在搜索引擎搜索“某设备开启开发者模式”，查看具体的开启教程。
    </div>

6. 单击 “Run” 按钮，编译并运行示例源码。
   ![/Pics/Android/ExpressSDK/SampleRunningGuide/run_android_demo_new.png](http://doc.oa.zego.im/Pics/Android/ExpressSDK/SampleRunningGuide/run_android_demo_new.png)

## 常见问题

1. **Android Studio 运行下载的示例源码时报错“错误：非法的表达式开始 :9”，如何处理？**

    ![/Pics/Android/ExpressSDK/SampleRunningGuide/as_run_android_demo_error_3.png](http://doc.oa.zego.im/Pics/Android/ExpressSDK/SampleRunningGuide/as_run_android_demo_error_3.png)

    由于下载的示例源码中，AppID ，UserID 和 Token 没有具体的值，请参考本文 “[3 运行示例源码](3125#1_3)” 章节中的步骤 5 进行处理。

2. **Android Studio 运行下载的示例源码时报错“错误：程序包im.zego.zegoexpress不存在”如何处理？**

    ![/Pics/Android/ExpressSDK/SampleRunningGuide/as_run_android_demo_error_4.png](http://doc.oa.zego.im/Pics/Android/ExpressSDK/SampleRunningGuide/as_run_android_demo_error_4.png)

    由于在中国大陆访问 GitHub 较慢，从 [Github 下载](https://github.com/zegoim/zego-express-example-topics-android) 的示例源码中并不包含 “ZegoExpressEngine” SDK，所以会报指定程序包不存在的问题，请参考本文 “[3 运行示例源码](3125#1_3)” 章节中的步骤 4 进行处理。
