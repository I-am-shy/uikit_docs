# 混流结束回调

---

## 1 描述

当开发者需要具体知道当前 App 的混流情况，可先 [配置回调\](!Callback/Configuration_Instructions)。App 在混流服务器有混流结束时，将会以 POST 的形式对回调地址进行请求。


<div class="mk-warning">


当出现如下情况：
- 混流结束请求处理失败时，没有回调。
- 混流过程中异常终止，没有回调。
</div>



## 2 回调说明

- 请求方法：POST。

@@@Server_API2_callback_intro_1@@@

- 请求地址：请联系 ZEGO 技术支持配置回调地址。
- 传输协议：HTTPS/HTTP，建议使用 HTTPS。

## 4 数据示例

```
{
    "appid": "111111",
    "channel_id": "2222",
    "event": "mix_stop",
    "mix_stream_alias": "aaa",
    "mix_url": "rtmp://rtmp.wsdemo.zego.im/zegodemo/aaa",
    "nonce": "7257787020413449571",
    "publish_id": "abc",
    "seq": "1234",
    "signature": "fadc2c4abb5714c30a711b3ad6f7783e37531263",
    "task_id": "111111:1111",
    "timestamp": "1689835223"
}
```

## 5 返回响应


返回 HTTP status code 为 2XX （例如 200）表示成功，其他响应都表示失败。
