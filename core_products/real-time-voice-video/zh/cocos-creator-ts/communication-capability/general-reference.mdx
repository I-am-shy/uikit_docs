### 4.3 使用 Token


用户携带获取到的 Token 和 user、roomID 信息，登录对应的房间。

@@@RTC_Token_LoginRoom@@@

```ts
let roomConfig = new ZegoRoomConfig()
roomConfig.token = 'xxxxxxxx' // 请求开发者服务端获取

// 登录房间
this.engine.loginRoom('your_room_id', new ZegoUser('user_id'), roomConfig)
```

如果开发者需要在用户登录房间后修改用户的推流权限，也可以调用 [renewToken\|_blank](@renewToken) 接口更新权限，更新后会影响下一次推流的权限，之前已经成功的推流不受影响。

```ts
let token = 'xxxxxxxx' // 重新请求开发者服务端获取 Token
this.engine.renewToken('your_room_id', token)
```

### 4.4 Token 过期时的处理方式

@@@Token_overdue@@@


在 Token 过期前 30 秒，SDK 会通过 [onRoomTokenWillExpire\|_blank](@onRoomTokenWillExpire) 回调发出通知。

收到该回调后，开发者需要从自己的服务端获取新的有效 Token，并调用 SDK 提供的 [renewToken\|_blank](@renewToken) 接口更新 Token。

@@@Token expiration processing mechanism@@@


```ts
onRoomTokenWillExpire(roomID: string, remainTimeInSecond: number): void {
  let token = 'xxxxxxxx' // 重新请求开发者服务端获取 Token
  this.engine.renewToken('your_room_id', token)
}
```
