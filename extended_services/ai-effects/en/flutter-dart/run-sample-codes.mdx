# Run the sample code

---

## Prerequisites

Please ensure your development environment meets the following requirements:

* Flutter 2.0 or above.
* iOS 13.0 or above and an iOS device or simulator that supports audio and video (real device recommended).
* Android 8.0 or above and an Android device or simulator that supports audio and video (real device recommended). For real devices, please enable "USB debugging".
* Device connected to the Internet.

## Requirements

Please ensure the following requirements are met before running:
* Registered an account on [ZEGOCLOUD Console](https://console.zegocloud.com/account/login) and obtained the AppID and AppSign required for SDK initialization.
* Contacted ZEGOCLOUD technical support to obtain the License file for beauty effects.

## Run the sample code

<Steps>
<Step title="Import resources and models">
Download the latest version of [iOS ZegoEffects SDK](/ai-effects-ios-objc/downloads) or [Android ZegoEffects SDK](/ai-effects-android-java/downloads), unzip it, and import the `Resources` and `Models` folders into the sample code folder you previously unzipped, and keep the original `Backgrounds.bundle` file.
<Tabs>
<Tab title="iOS">
Place the `Resources` and `Models` folders in the `Resources` folder, and after importing, your project directory structure should look like this:
```bash
Runner
├── Resources
│   ├── Models
│   │   ├── FaceDetectionModel.model
│   │   └── SegmentationModel.model
│   └── Resources
│       ├── Backgrounds.bundle
│       ├── ClarityResources.bundle
│       ├── ColorfulStyleResources
│       ├── CommonResources.bundle
│       ├── FaceWhiteningResources.bundle
│       ├── MakeupResources
│       ├── PendantResources.bundle
│       ├── RosyResources.bundle
│       ├── SkinColorResources
│       └── TeethWhiteningResources.bundle
│
```
</Tab>
<Tab title="Android">
Add the `Resources` and `Models` folders to the `assets` directory of your sample code. Usually, the resources should be placed in the `android/app/src/main/assets` directory, so after importing, your sample code project directory structure should look like this:
```bash
android/app/src/main
├── AndroidManifest.xml
├── assets
│   ├── Models
│   │   ├── FaceDetectionModel.model
│   │   └── SegmentationModel.model
│   └── Resources
│       ├── Backgrounds.bundle
│       ├── ClarityResources.bundle
│       ├── ColorfulStyleResources
│       ├── CommonResources.bundle
│       ├── FaceWhiteningResources.bundle
│       ├── MakeupResources
│       ├── PendantResources.bundle
│       ├── RosyResources.bundle
│       ├── SkinColorResources
│       └── TeethWhiteningResources.bundle
├── java
└── res
```
</Tab>
</Tabs>
</Step>
<Step title="Configure authentication information">
In the root directory of the sample code, find the `zego_config.dart.template` file, remove the "template" suffix, and fill in the corresponding `appID` and `appSign`. For details, see [Requirements](#requirements).

```dart
// Login to the console https://console.zegocloud.com to get appID, for example const appID = 123456789;
int appID = 0;  
// Login to the console https://console.zegocloud.com to get appSign, for example "39011cbxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
String appSign = "";
```
</Step>
<Step title="Get dependencies">
Run the following command in the root directory to pull the Flutter dependencies.
```bash
flutter pub get
```
</Step>
<Step title="(For iOS) Run the pod command">
Run the following command in the iOS directory.
```bash
pod install
```
</Step>
<Step title="Run the sample code">
For Android, click the `Run` button in the upper right corner of Android Studio to run the sample app.

</Step>
</Steps>
